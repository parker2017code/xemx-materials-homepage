# xemX Copy Style

This repo owns the public homepage for xemX materials space exploration. Public copy should sound like a materials R&D partner, not generic SaaS or AI-platform prose.

## Sentence Standard

Keep a sentence when it adds one of these:

- material system
- target property
- sample plan
- measured output
- wafer position or composition range
- decision the reader can make
- constraint, cost, timing, or next validation step
- useful judgment about fit or risk

Cut the sentence when a generic software writer could have used it without knowing materials R&D.

## xemX Voice

Start with the experiment, then name the service.

Good xemX copy says what changes for an R&D team: fewer blind formulation guesses, more related samples in one campaign, measured property maps, and a clearer next validation run.

Weak copy talks about a platform in broad terms. Replace it with the material question, sample count, measurement, map, and decision.

## Public Claim Rule

Public homepage claims need approval when they involve:

- numbers
- facility access
- partner-supported routes
- standard deliverables
- timelines
- performance promises
- customer outcomes

The safe public pattern is:

> xemX tests [real material variants] to measure [property or failure mode] before [expensive validation step].
