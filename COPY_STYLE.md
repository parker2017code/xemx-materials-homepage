# xemX Copy Style

Use this file for xemX-facing repo text, docs, prompts, PR text, commit messages, and any copy destined for preview or public pages.

## Brand

- Write the brand as `xemX` in prose.
- Preserve existing logo files and graphic assets.
- Do not change visible copy on the public homepage at `https://xemx-materials.com/` unless the user explicitly asks for that page.

## Sentence Standard

Every sentence should add at least one useful thing:

- material question
- sample or library detail
- measured output
- reviewer decision
- constraint, caveat, or failure mode
- next validation step

If a sentence adds none of those, cut it or rewrite it.

## Voice

Write like a materials team explaining work to a technical reviewer. Prefer plain verbs: make, measure, compare, screen, validate, reject, repeat.

Replace praise words with the measurement, sample detail, workflow step, or reviewer decision they hide.

## Claim Boundary

Do not make a campaign claim broader than the proof point. A measured map can narrow candidate regions. It does not prove full device survival, customer validation, production approval, or manufacturing transfer unless that evidence is present.
